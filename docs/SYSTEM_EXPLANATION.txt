IMEDSTOCK 2.0 - SYSTEM EXPLANATION (TAGLISH)
Date generated: 2026-03-23

============================================================
1) OVERVIEW NG SYSTEM
============================================================
Ang iMedStock 2.0 ay Laravel 10 based web app para sa medical shop operations.
Main capabilities:
- Product inventory management (items + stock)
- Service catalog and service checkout
- Cart and checkout flow for products and services
- User profiles and customer details
- Admin dashboard (charts + data tables)
- Reviews with profanity masking
- Email notifications for order status
- Search with filters (price/category/brand/service type)

Tech stack summary:
- Backend: PHP 8.1+, Laravel 10
- Frontend build: Vite + Bootstrap + Sass
- Charts: consoletvs/charts
- Tables: yajra/laravel-datatables + buttons
- Import: maatwebsite/excel
- PDF/email support: dompdf + Laravel Mail

Core dependency sources:
- composer.json
- package.json

8) CHECKOUT AND ORDER LIFECYCLE
============================================================
Product checkout lifecycle:
1. User adds item(s) to cart
2. System validates stock and quantity
3. User opens checkout, picks payment method, optional discount code
4. System creates orderinfo row
5. System inserts orderline rows
6. System deducts stock quantities
7. System sends order email (best effort)
8. Cart is cleared/restored depending on buy-now mode

Service checkout lifecycle:
1. User buy-now or adds service(s) to service cart
2. User enters payment + schedule (date/time/mode)
3. System creates orderinfo row with schedule fields
4. System inserts service_orderline rows
5. System sends order email (best effort)
6. Service cart is cleared

Admin update lifecycle:
1. Admin opens order detail
2. Admin updates status
3. System sends updated status email to customer

13) SESSION AND CART DESIGN
============================================================
Product cart:
- Session key is user-scoped (cart_user_<id>)
- Supports buy-now backup/restore flow

Service cart:
- Session key is user-scoped (service_cart_user_<id>)
- Independent from product cart session

Impact:
- Product and service checkout are separated by cart namespace

15) IMPORTANT NOTES / DESIGN OBSERVATIONS
============================================================
- System supports both physical product sales and service booking in one platform.
- Review posting is protected by verified-purchase rule (delivered status).
- Order amount calculations include fallback expressions for mixed item/service orders.
- Admin dashboard is heavily query-driven and supports date-range filtering.
- Bad-word filtering is centralized and reusable.
- Normalization work removed duplicated customer data from order table, so customer details are resolved via joins/relations.
